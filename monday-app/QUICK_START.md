## ‚ö° 5-Minute Deployment

### 1. Fork & Deploy
```bash
# Fork this repository on GitHub first
# Then deploy to Render with one click:
# https://render.com/deploy?repo=YOUR_GITHUB_REPO_URL
```

### 2. Configure Environment Variables
In Render Dashboard, add these environment variables:

**Required:**
- `MONDAY_CLIENT_ID`: Get from Monday Developer Portal
- `MONDAY_CLIENT_SECRET`: Get from Monday Developer Portal  
- `MONDAY_APP_ID`: Your Monday App ID
- `MONDAY_SIGNING_SECRET`: Webhook signing secret from Monday

**Optional (auto-generated):**
- `JWT_SECRET`: Auto-generated by Render
- `WEBHOOK_SECRET`: Auto-generated by Render

### 3. Update Monday Developer Portal
1. Go to [Monday Developer Portal](https://monday.com/developers)
2. Open your app settings
3. Update **Base URL**: `https://your-app-name.onrender.com`
4. Update **Redirect URI**: `https://your-app-name.onrender.com/auth/callback`
5. Add webhook endpoints:
   - `https://your-app-name.onrender.com/api/webhooks/item/create`
   - `https://your-app-name.onrender.com/api/webhooks/status/change`

### 4. Test Installation
1. In Monday Developer Portal, click "Install App"
2. Select a workspace to install
3. Open any board ‚Üí View ‚Üí "Automation Dashboard"
4. You should see the automation interface!

## üêõ Troubleshooting

### Common Issues

**"App not found" error**
- Check if Render deployment is live
- Verify base URL in Monday Developer Portal

**OAuth redirect fails**
- Ensure redirect URI matches exactly
- Check that app is published in Monday

**Webhooks not working**
- Verify webhook secrets match
- Check Render logs for webhook processing

**Blank dashboard**
- Check browser console for errors
- Verify Monday SDK is loading

### Debug Mode
Enable debug logging:
```bash
# In Render environment variables
ENABLE_DEBUG_MODE=true
LOG_LEVEL=debug
```
